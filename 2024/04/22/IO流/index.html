文件是计算机硬盘中非常重要的存储方式

File

File是java.io包下的类，File类的对象，用于代表当前操作系统的文件(文件或文件夹)

File类只能对文件本身进行操作(获取文件信息、判断文件类型、创建删除等)，不能读写文件里存储的数据

当File对象指向的是文件夹时，length返回值为文件夹的大小，不包含文件夹中的内容

常用方法

文件类型信息操作：

  1. file.exists()：判断当前文件对象对应的文件路径是否存在 ，存在返回true
  2. file.isFile()：判断当前文件对象指代的是否为文件 ，是文件返回true
  3. file.isDirectory()：判断当前文件对象指代的是否为文件夹 ，是文件夹返回true
  4. file.getName()：获取文件的名称，包括后缀
  5. file.length()：获取文件的大小，返回字节个数
  6. file.lastModified()：获取文件的最后修改时间
  7. file.getPath()：获取创建文件对象时使用的路径
  8. file.getAbsolutePath()：获取绝对路径

创建、删除文件：

  1. file.createNewFile()：不存在的文件路径创建对象调用方法新建空文件，创建成功返回true
  2. file.mkdir()：只能创建一级文件夹，创建成功返回true
  3. file.mkdirs()：可以创建多级文件夹，创建成功返回true
  4. file.delete()：删除文件或者空文件夹，删除成功返回true

遍历文件夹：

  1. file.list()：返回字符串数组，内容为一级文件名称
  2. file.listFiles()：返回当前目录下所有一级文件对象到对象数组中

如果对象是文件或不存在路径会返回null，空文件夹返回长度为0的数组，没有权限也会返回null

字符集

常用字符集

  1. ASCII码：美国指定的一套字符编码(1字节)，规定了英语字符和二进制之间的关系，最前面一位统一规定0，只用后七位表示了128个字符
  2. Unicode：国际组织制定的统一码，万国码，一种编码包含世界上所有的符号，每个符号都有独一无二的编码，所以不会乱码，一个英文字母和汉字都是2字节，浪费空间
  3. UTF-8：是互联网上使用最广的一种Unicode实现方式，可以使用1-6个字节表示一个符号，根据不同符号变化字节长度，字母占1个字节，汉字占3个字节
  4. GBK：字母数字1字节，汉字2字节且汉字的第一个字节的第一位为1，表示的汉字比GBK2312多

释放资源的方式

  1. try-catch-finally：无论try中的程序是否正常一定会执行finally区除非JVM终止，可以将流关闭放在finally块中，但是显得比较臃肿
  2. try-with-resource：把用到的流资源放在try后面的参数小括号中**(只能放资源对象)**的try-catch语句，结束后会自动调用流的close方法将括号中的流释放

资源：资源一般指最终实现了AutoCloseable接口

**字符输出流写出数据后必须刷新流或者关闭流，写出去的数据才能生效！！！**因为会将所有要写入的字符加载到缓冲区中，然后再进行一次调用系统写入以提高效率，通知缓冲区调用系统需要刷新流或者关闭流(包括刷新流操作)，如果缓冲区被 填满的话会自动刷新

复制文件时使用缓冲流按照字节数组形式复制是速度最快的，字符流可以拷贝文本文件，但不可以拷贝非文本文件，因为字符流在拷贝文件过程中，就先会将读取到的数据转换为字符，而非文本文件内容本质上并不是一个字符，因此要转换为对应的字符会出现乱码，而最终将乱码写入到新的文件中，造成新的文件损坏；

IO转换流

如果代码的编码和被读取的文本文件的编码不一致，使用字符流读取文本文件时就会出现乱码

字符输入转换流

得到原文件的字节流按照指定字符集编码转换为字符输入流，再将字符输入流包装成缓冲字符输入流

字符输出转换流

获取字节输入流，再按照指定的字符集编码将其转换成字符输入流，以后写出去的字符就会用该字符集编码

IO打印流

PrintStream/PrintWriter，打印流可以更方便更高效的打印数据出去，能实现打印什么就是什么

PrintStream和PrintWriter的打印用法相同，写有区别且都无法进行追加，PrintStream继承自字节输出流OutputStream因此支持写字节数据的方法，PrintWriter继承自字符输出流Writer，因此支持写字符数据

DataOutputStream

允许将数据和其类型一并写出去，通过创建一个数据输出流包装低级字节输出流，存进去看起来跟乱码一样，存储不是为了查看，而是为了读取

DataInputStream

用于读取数据输入流写出来的数据

IO序列化流

对象序列化：把Java对象写入到文件中去，相对应的为对象反序列化

ObjectOutputStream

注：类必须实现Serializable接口后对象才能实现序列化

ObjectInputStream

如果在类的属性定义时加入transient则该成员变量不再参与序列化

如果要一次序列化多个对象，可以使用ArrayList集合存储多个对象，然后直接对整个集合进行序列化即可，ArrayList集合已经实现了序列化接口
