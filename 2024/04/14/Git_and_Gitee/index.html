Git_and_Gitee

Git 是一个开源的分布式版本控制系统，用于敏捷高效地处理任何或小或大的项目，是为了帮助管理 Linux 内核开发而开发的一个开放源码的版本控制软件。

Git常用命令

设置用户签名：

  1. git config --global user.name xxx：指定用户
  2. git config --global user.email xxx：指定邮箱

用户签名会记录在当前用户文件的.gitconfig中，只需要设置这一次

初始化本地库：每个项目都要执行一次git init，会生成一个隐藏的.git配置文件夹

查看本地库状态：项目(工作区workspace)中新创建的文件处于untracked(未跟踪)状态，已有的文件修改后处于unstaged(未暂存状态)，通过git status可以查看文件的状态

添加/移除暂存区：

  1. git add 文件名可以将工作区的文件移动到暂存区(index)

  2. git add .可以全部移到暂存区

  3. git rm --cached 文件名可以将文件从暂存区退回到工作区

提交本地仓库：git commit -m '日志信息'

查看日志：git log查看当前日志或者git reflog查历史日志，历史日志是所有日志，当前日志不显示后面版本

版本切换：使用git reset --hard 版本号可以切换版本，版本号为日志文件的commit

Git分支操作

查看分支：git branch -v

创建分支：git branch 分支名，相当于对上一个分支完整的克隆，包括所有日志

切换分支：git checkout 分支名

创建并切换分支：git checkout -b 分支名

删除分支：git branch -d 分支名删除时需要做检查，而且不能删除当前分支，D为强制删除不检查

正常合并分支：git merge 分支名把指定的分支合并到当前分支上，一般将分支合并到master

冲突合并分支：提交之后发生冲突，要手动处理文件中的冲突然后再加入暂存区提交到仓库

Git分支开发规范

master：用于部署生产环境的分支，确保master分支的稳定性，一般由develop和hotfix分支合并，任何时间都不能直接修改代码

hotfix：线上出现紧急问题需要及时修复，以master分支为基线创建hotfix分支，修复完成后要合并到master和develop分支，命名为hotfix/user

develop：始终保持最新完成以及bug修复后的代码，feature分支都是基于develop分支下创建的

feature：开发新功能时，以develop为基础创建feature分支，命名为feature/user_module

release：一组feature开发完成时首先合并到develop分支进行测试发布时会创建release分支，测试如果发现了bug由develop需要改bug后重新合并，测试通过后合并到master分支

Gitee远程仓库

产生公钥：ssh-keygen -t rsa

查看公钥：cat ~/.ssh/id_rsa.pub

验证公钥：ssh -T git@gitee.com

添加远程仓库：git remote add origin(仓库名) 仓库地址

查看远程仓库：git remote

推送远程仓库：git push origin master:master

建立关联：

  1. git push --set-upstream origin master
  2. git branch -vv
  3. git push之后更新只要直接运行3就可以上传

克隆：git clone 仓库路径 本地目录

拉取：git pull origin master
