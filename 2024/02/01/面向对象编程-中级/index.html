包、封装、继承、多态以及Object类的使用细节

包

作用：区分相同名字的类，方便管理类，控制访问范围
本质：创建不同的文件夹来保存类文件

细节：package 的作用是声明当前类所在的包，需要放在类的最上面且一个类只能有一句 package

包的命名

规则

只能包含数字、字母、下划线、圆点，不能数字开头，不能是关键字或保留字

规范

com.公司名.项目名.业务模块名
com.sina.crm.user//用户模块

访问修饰符

java 提供了四种访问控制修饰符号，用于控制方法和属性的访问权限
同一类 < 同一包 < 子类 < 不同包

  1. 公开级别：用public修饰，对外公开
  2. 受保护级别：用protected修饰，对子类和同一个包中的类公开
  3. 默认级别：没有修饰符，向同一个包的类公开
  4. 私有级别：用private修饰，只有类本身可以访问，不对外公开

面向对象的 三大特征

封装

封装可以隐藏实现细节，可以对数据进行验证来保证其安全合理
将属性私有化，然后提供 set 和 get 方法来操作
构造器可以调用 set 方法实现对属性的保护

继承

继承可以解决代码复用，使编程更加接近人类思维
当多个类存在相同的属性(变量)和方法时，可以从这些类中抽象出父类，在父类中定义这些相同的属性和方法，所有的的子类不需要重新定义这些属性和方法，只需要通过 extends 来声明继承父类即可
父类又叫超类、基类；子类又叫派生类

  1. 子类继承了所有的属性和方法，但是私有属性和方法不能在子类直接访问，要通过公共的方法去访问
  2. 子类必须调用父类的构造器，完成父类的初始化
  3. 创建子类对象时，不管子类使用哪个构造器，默认情况下总会去调用父类的无参构造器，如果父类没有无参构造器，则必须在子类构造器中用 super 去指定使用父类的某构造器完成父类的初始化
  4. 如果用 super 指定调用父类的某构造器，要显式的调用一下
  5. super 使用时必须放在构造器的第一行
  6. super()和 this()都只能放在构造器第一行，所以两个方法不能共存
  7. java 所有类都是 Object 的子类
  8. java 是单继承机制，子类最多只能继承一个父类(直接继承)

this 和 super 语句在构造器中只能存在一个，因此 B 的有参构造器默认有一句super();
所以输出结果为a,b name,b,

方法重写

  1. 方法重写(override)也叫方法覆盖，需要子类方法的参数和方法名称与父类的完全一样
  2. 子类的返回类型要和父类一样或者是父类返回类型的子类，比如父类public Object getInfo(){}，子类可以是public String getInfo(){}
  3. 子类方法不能缩小父类方法的访问权限

多态

方法或对象具有多种形态
多态建立在封装和继承之上，重写和重载就是多态的体现
多态的前提是：两个对象(类)存在继承关系

对象的多态(重点)

  1. 一个对象的的编译类型和运行类型可以不一致
  2. 编译类型在定义对象时就确定下来不能改变，运行类型可以改变，getClass()可以查看运行类型

向上转型

本质：父类的引用指向了子类的对象(如上 Dog 例子)

  1. 可以调用父类的所有成员(需遵循访问权限)
  2. 不能调用子类的特有成员(属性不能重写，可以理解为特有的方法和所有属性)(因为编译阶段调用的成员由编译类型决定)
  1. 最终运行效果看子类(运行类型)的具体实现

向下转型

向下转型后可以调用子类类型中所有的成员

  1. 只能强制转换父类的引用，不能强制转换父类的对象
  2. 父类的引用必须指向的是当前目标类型的对象(如上 animal 的运行类型为 cat)
  3. Dog 和 Cat 都是 Animal 的子类，但是 Dog 和 Cat 之间是没有关系的(不能转换)

instanceof 比较操作符

用于判断对象的运行类型是否为某类型或者某类型的子类型 aa instanceof AA//返回布尔类型值

动态绑定机制

  1. 调用对象方法的时候，方法会和该对象的内存地址(运行类型)绑定
  2. 当调用对象属性时，没有动态绑定机制，哪里声明哪里使用

Object 类

equals 方法

  1. ==既可以判断基本类型又可以判断引用类型，引用类型判断的是地址是否相等，即判断是不是一个对象
  2. Object 类中的 equals 只能判断引用类型，默认判断的是地址是否相等，子类中往往重写成判断内容是否相等(比如 Integer,String 等中的 equals)

hashCode 方法

  1. 提高具有哈希结构的容器的效率
  2. 两个引用如果指向的是同一对象，哈希值一定相等
  3. 哈希值主要是根据地址号计算得来的，不能完全将哈希值等价于地址

toString 方法

  1. Object 类中默认返还全类名+@+十六进制哈希值，子类一般重写返还对象属性信息
  2. 直接输出一个对象会默认调用 toString()

finalize

当某个对象没有任何引用的时候，jvm 就会认为这个对象是垃圾，就会使用垃圾回收机制销毁该对象，销毁对象前会调用 finalize 方法。子类可以重写该方法做一些释放资源的操作，垃圾回收机制的调用是由系统自己决定的，也可以通过 System.gc()主动触发垃圾回收机制
