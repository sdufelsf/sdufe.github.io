String

String对象用于保存字符串，字符串常量对象是用双引号括起来的字符序列，字符串的字符使用Unicode字符编码，一个字符(不区分字母汉字)占两个字节

结构

继承了三个接口，CharSequence，Comparable(compareTo)，Serializable(序列化)

String创建的机制

String直接赋值String str = "hello"会先在常量池中寻找是否有要存储的字符串，有的话栈里的对象名会直接指向常量中的字符串地址，如果没有就会在常量池创建该字符串

String调用构造器创建String str = new String("hello")会在堆中创建一个由栈中对象名指向的对象，对象中的value数组会指向常量区中的字符串地址，如果常量池中有则直接指向，没有则需要创建

String特性

String是final类，底层代码的本质是一个字符串数组private final char value[];其中value是final类是指其地址不可修改，而不是内容，修改String对象的本质是先在常量池中寻找，没有找到就在常量池中建立一个新的字符串，然后让String对象指向新的字符串

常量相加看常量池，变量相加看的是堆

String a = "hel" + "lo";优化等价String a = "hello";编译器会判断创建的常量池对象是否有指向，

equals 区分大小写判断内容是否相等

equalsIgnoreCase 忽略大小写判断内容是否相等

length 获取字符个数，字符串长度

lastIndexOf 获取字符在字符串中最后一次出现的索引，索引从0开始，没找到返回 -1

String的缺点

String类是保存字符串常量的，每次更新都需要重新开辟空间，效率比较低，因此设计了StringBuilder和StringBuffer来增强String的功能并提高效率

StringBuffer

代表可变的字符序列，可以对字符串内容进行增删，很多方法和String相同，但StringBuffer是可变长度的，实质是父类中的一个存放在堆中的字符数组char[] value;

结构

StringBuffer是一个final类不能被继承，直接父类是AbstractStringBuilder，实现了Serializable，即StringBuffer的对象可以串行化(对象可以网络传输，也可以保存到文件)

无参构造器默认长度16，有参数+16

String和StringBuffer转换

String -> StringBuffer

String str = “hello”

StringBuffer sb1 = new StringBuffer(str); //构造器返回的才是对象，对str无影响

StringBuffer sb2 = new StringBuffer();

sb2 = sb2.append(str); //构造空对象使用append方法

StringBuffer -> String

String s = sb1.toString(); //使用包内提供的toString方法

String s1 = new String(sb1) //使用构造器

StringBuffer常用方法

append 在后面追加字符串，直接修改源字符串

delete 两个数删除左闭右开区间的字符

replace 两个数字和字符串，左闭右开替换

indexOf 查找指定的字串第一次出现的索引，如果找不到返回 -1

insert 一个索引和一个字符串，将字符串插入后原索引位置的内容自动后移

length 返回长度

StringBuilder

可变的字符序列，结构与方法和StringBuffer一样，但是不能保证同步(StringBuilder不是线程安全)。

该类被设计用作StringBuffer的一个简易替换，用在字符串缓冲区被单个线程使用的时候。如果可以，优先采用该类，因为大多数实现中StringBuilder比StringBuffer要快

总结

  1. String是不可变序列，效率低但是复用率高
  2. StringBuffer是可变字符序列，效率较高，线程安全
  3. StringBuilder是可变字符序列，效率最高但是线程不安全
  4. 如果要对字符串做出大量修改不要使用String
