集合

2024-03-09
Word count: 2.2k   |   Reading time≈ 8 min

为了方便操作多个对象，那么我们就得把这多个对象存储起来，想要存储多个对象(变量),很容易就能想到一个容器(集合)来装载，简单来说集合就是“由若干个确定的元素所构成的整体”

集合框架体系

  1. Collection接口

    1. List接口

      1. ArrayList

      2. Vector

      3. LinkedList

    2. Set接口

      1. HashSet
      2. TreeSet
      3. LinkedHashSet

    3. Queue接口

      1. ArrayBlockingQueue
      2. LinkedBlockingQueue
      3. SynchronousQueue
      4. PriorityBlockingQueue

  2. Map接口

    1. HashMap
    2. LinkedHashMap
    3. Hashtable
    4. TreeMap

Collection

实现子类可以存放多个元素，每个元素都可以是Object类，Collection接口没有直接实现的子类

接口实现子类的常用方法有add(添加单个元素)，remove(删除指定元素)，contains(查找元素)，size(获取元素个数)，isEmpty(判断是否为空)，clear(清空)，addAll(添加多个元素)，containsAll(查找多个元素)，removeAll(删除多个元素)

Collection接口的遍历

  1. 使用Iterator(迭代器)，所有实现了Collection接口的集合类都有一个iterator方法返回一个迭代器

  2. 增强for循环(底层还是迭代器)

List

List集合类中元素有序(添加顺序和取出顺序一致)可重复，每个元素都有对应的顺序索引

add(index, obj); 在索引index位置插入，没有index默认放最后

addAll(index, list2); 直接插入另一个list的全部元素

get方法可以获得索引位置的元素

indexOf返回元素在当前集合中首次出现的位置

lastIndexOf返回元素在当前集合中末次出现的位置

remove移除指定index位置的元素并返回此元素

set(index, obj)将index位置的元素换为obj，相当于替换

subList(start, end);返回区间的子集合

ArrayList

可以存放各种元素，包括null而且是多个，是由数组实现数据存储的
基本等同Vector，除了线程不安全，多线程情况下不建议使用ArrayList

底层操作机制：

  1. ArrayList中维护了一个Object类型的数组elementData
  2. 无参构造器创建ArrayList对象则elementData初始值为0，创建了一个空数组，elementData大小不够时就调用grow方法扩容，第一次添加扩容为10，再需要扩容则1.5倍
    1. 初始为0在grow方法中会先将 minCapacity(10)赋给newCapacity
    2. 不为0的时候执行newCapacity = oldCapacity + (oldCapacity >> 1);
    3. 最后使用elementData = Arrays.copyOf(elementData, newCapacity);赋值
  3. 指定大小构造器创建则初始elementData容量为指定大小，扩容则1.5倍

Vector

Vector底层也是一个对象数组，操作方法都有synchronized，是线程同步的

无参构造器默认调用有参构造器大小10，扩容是变为2倍

LinkedList

底层实现了双向链表和双端队列的特点，可以添加任意重复元素，包括null，但是线程不安全，没有实现同步

选择

改查操作多选ArrayList，增删操作多选LinkedList
根据业务灵活选择可能一个模块使用ArrayList另一个模块用LinkedList

Set

无序(添加和取出的顺序不一致)，没有索引，不能重复元素，只能含有一个null

没有get方法，不能用普通for循环索引遍历

HashSet

实际上是HashMap(数组+链表+红黑树)，不保证元素有序，取决于hash后的结果转成的索引值

先取得元素的哈希值(hashCode方法)，对哈希值进行运算得到一个索引值即为要存放在哈希表中的位置号，如果该位置上没有其他元素则直接存放，如果已经有其他元素则需要进行equals(每个类都有自己对应的equals)判断，如果相等则不再添加，不相等就以链表形式添加

扩容机制：第一次添加时，table数组扩容到16，临界值(threshold)16*加载因子(loadFactor)0.75=12，如果table数组使用到临界值12，就会扩容到16*2=32，新的临界值就是32*2=32，新的临界值就是32*0.75=24，以此类推，临界值包含table里面的所有元素，链表上的每个元素都算

Java8中，如果链表的元素个数不小于TREEIFY_THRESHOLD(8)且table不小于MIN_TREEIFY_CAPACITY(64)，就会进行树化(红黑树)

LinkerHashSet

底层是一个LinkedHashMap(HashMap的子类)，维护了一个数组+双向链表(LinkedHashSet有head和tail)，每个节点有before和after属性，取出的顺序和插入的顺序相同，在根据hashCode决定元素存储位置的同时用链表维护了元素的次序

Map

JDk8的Map接口特点：

  1. Map用于保存具有映射关系的数据：Key-Value(双列元素)
  2. Map中的key和value可以是任何引用类型的数据，会封装到HashMap$Node对象中
  3. Map中的key不允许重复(HashSet)，key相同时等价于替换，value可以重复
  4. key和value之间存在单向一对一关系，通过指定的key总能找到对应的value
  5. Node实现了Entry接口，所以可以说一对k-v就是一个Entry
  6. k-v为了方便遍历还会创建EntrySet集合存放Entry类型元素，Entry对象对应一个k-v
  7. entrySet里定义的类型为Map.Entry，实际上存放的还是HashMap$Node
  8. 因为Map.Entry提供了getKey()和getValue()方法，所以方便我们遍历

常用方法

put 添加

remove 根据键删除映射关系

get 根据键值获取值

size 获取元素个数

isEmpty 判断个数是否为0

clear 清除所有键值对

containsKey 查找键是否存在

HashMap

是Map接口使用频率最高的实现类，不保证映射顺序，因为底层是以hash表的方式存储的
HashMap没有实现同步，因此是线程不安全的

Hashtable

键和值都不允许为null，使用方法和HashMap一样，但是hashTable的线程是安全的
底层数组初始为11，加载因子也为0.75，扩容方法为2倍加1

Properties

继承自Hashtable类并实现了Map接口，使用特点和Hashtable类似，还可以用于从xxx.properties文件中 加载数据到Properties类对象并进行读取和修改

选择集合实现类

  1. 先判断存储的类型(一组对象[单列]或一组键值[双列])
  2. 一组对象：Collection接口
    1. 允许重复：List
      1. 增删多：LinkedList[底层维护了一个双向链表]
      2. 改查多：ArrayList[底层维护Object类型的可变数组]
    2. 不允许重复：Set
      1. 无序：HashSet[底层是HashMap，维护了一个哈希表(数组+链表+红黑树)]
      2. 排序：TreeSet
      3. 插入和取出顺序一致：LinkedHashSet(数组+双向链表)
  3. 一组键值对：Map
    1. 键无序：HashMap[底层是哈希表，数组+链表+红黑树]
    2. 键排序：TreeMap
    3. 键插入和取出顺序一致：LinkedHashMap
    4. 读取文件：Properties

Collections

是一个提供Set、List、Map等集合的工具类，提供了一系列静态的方法对集合元素进行排序、查询和修改等操作

排序操作(均为static方法)：

  1. reverse(List)：反转List中元素的顺序
  2. shuffle(List)：对List集合元素进行随机排序
  3. sort(List)：根据元素的自然顺序对指定List集合元素按升序排列
  4. sort(List, Comparator)：根据指定的Comparator产生的顺序对List集合元素排序
  5. swap(List, int, int)：将指定的list集合中的i处元素和j处元素进行交换

查找、替换

  1. Object max(Collection)：根据元素的自然顺序，返回给定集合中的最大元素
  2. Object max(Collection, Comparator)：根据Comparator指定的的顺序返回给定集合中的最大元素
  3. Object min(Collection)
  4. Object min(Collection, Comparator)
  5. int frequency(Collection, Object)：返回指定集合中指定元素的出现次数
  6. void copy(List dest, List src)：将src中的内容复制到dest中，拷贝前要给dest赋值使其和list.size()一样大
  7. boolean replaceAll(List list, Object oldVal, Object newVal)：使用新值代替所有旧值
